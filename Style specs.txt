1. Color System

Define CSS variables matching the official palette
:

:root {
  /* Primary background and dark surfaces */
  --color-caja-fuerte: #101021;   /* RGB 16,16,33 — Pantone Black 6 C:contentReference[oaicite:1]{index=1} */
  --color-firma-azul:   #131a4e;   /* Deep blue (RGB 19,26,78 — Pantone 2766 C):contentReference[oaicite:2]{index=2} */
  /* Accent colors */
  --color-red-electrica:   #422BE2; /* Electric blue/purple — Pantone 2368 C:contentReference[oaicite:3]{index=3} */
  --color-alerta:         #f93243; /* Vibrant red (RGB 249,50,67 — Pantone Red 032 C):contentReference[oaicite:4]{index=4} */
  --color-billete-nuevot: #9bd1b8; /* Mint green (RGB 155,209,184 — Pantone 344 C):contentReference[oaicite:5]{index=5} */
  --color-saldo-claro:    #e7e9e4; /* Soft off‑white (RGB 231,233,228 — Pantone 663 C):contentReference[oaicite:6]{index=6} */
  /* Neutrals */
  --color-white:     #ffffff;
  --color-black:     #000000;
}


Use caja‑fuerte and firma‑azul as primary backgrounds, alternating for contrast (e.g., dark hero backgrounds, navy cards).

Use alerta for primary call‑to‑action buttons and highlights.

Use red‑electrica and billete‑nuevot as secondary accents or highlight bars.

Saldo‑claro should be reserved for light backgrounds and subtle containers.

Follow the guide’s recommended proportions (approx. 30% deep blue, 20% red, 10% mint, 10% dark) to maintain balance
.

2. Typography

Primary font: Bricolage Grotesque (Google Fonts).

Use Medium weight for H1/H2 headings.

Use Extra Light for secondary headings and large supporting text
.

Secondary font: Roboto for paragraphs and UI labels
; default to Roboto Regular/Light for body copy to improve readability.

CSS examples:

body { font-family: "Roboto", sans-serif; line-height: 1.6; }
h1, h2 { font-family: "Bricolage Grotesque", sans-serif; font-weight: 500; }
h3, h4 { font-family: "Bricolage Grotesque", sans-serif; font-weight: 300; }


Establish a clear typographic hierarchy; bigger sizes (clamp functions) for hero titles, medium sizes for section headings, and smaller sizes for captions.

Keep adequate spacing between headings and paragraphs (use the spacing tokens below).

3. Spacing & Layout Tokens

Adopt an 8‑point grid: define variables like:

:root {
  --space-xs: 8px;
  --space-sm: 16px;
  --space-md: 24px;
  --space-lg: 40px;
  --space-xl: 64px;
}


Use max‑width: 1200px for content containers; center them with auto margins.

Vertical section spacing: at least var(--space-xl) between major blocks.

Horizontal gutters: var(--space-md) or var(--space-lg) depending on layout.

4. Shapes & Modules

Emulate Insular’s distinctive module shape: a rounded rectangle whose top‑left and bottom‑right corners are diagonally rounded
.

Example CSS class:

.module {
  border-radius: 0 24px 0 24px;
  background-color: var(--color-firma-azul);
  padding: var(--space-lg);
}


Use these modules for cards, callouts, and image containers to reinforce brand consistency.

5. Iconography & Illustrations

Design icons with thin, consistent strokes and subtle rounded edges, starting from geometric shapes
.

Icons should avoid excessive detail to remain legible at small sizes; include tiny “micro‑gestures” (e.g., a subtle tilt or curve) to give personality
.

For color, fill icons with alerta (red) or red‑electrica (purple/blue) and use billete‑nuevot for secondary details.

Illustrations should feature dramatic contrasts and warm/cool transitions to convey emotion, staying within the brand palette
.

6. Photography Style

Use natural, documentary‑style photos: candid moments, soft lighting, and organic compositions
.

Emphasize real human interactions (family connections, everyday scenes) and subtle “magic moments” (e.g., beams of light, reflections)
.

Avoid stiff poses; images should feel genuine and inclusive.

7. Buttons & CTAs

Primary button:

.btn-primary {
  background-color: var(--color-alerta);
  color: var(--color-white);
  border-radius: 32px;
  padding: 14px 32px;
  font-weight: 500;
  transition: transform 0.25s, box-shadow 0.25s;
}
.btn-primary:hover {
  box-shadow: 0 8px 20px rgba(0,0,0,0.2);
  transform: translateY(-2px);
}


Secondary button: outline variant with border color firma‑azul or billete‑nuevot; invert colors on hover.

All buttons must have a minimum hit area of 44×44 px and visible focus states (e.g., outline ring).

8. Cards & Containers

Cards should use soft shadows (box-shadow: 0 4px 16px rgba(0,0,0,0.15)) and the module corner style.

Base card background can be caja‑fuerte or firma‑azul, with contrasting text.

Use CSS variables for border radius (e.g., var(--radius-lg): 24px) to keep consistent corner sizes.

9. Animations with GSAP

Entrance animations: use gsap.from() with small vertical offset and fade‑in for sections (e.g., y: 40, opacity: 0).

ScrollTrigger: animate modules as they enter the viewport (e.g., color blocks slide up or expand).

Counters: animate currency rates with gsap.utils.interpolateNumber.

Micro‑interactions: button hover lifts; icons pulse subtly.

Always respect prefers-reduced-motion by disabling non‑essential animations when the user opts out.

10. Accessibility & Best Practices

All colors must achieve sufficient contrast (WCAG AA). For example, white text on alerta (#f93243) and firma‑azul (#131a4e) is acceptable, but test each combination.

Provide aria-labels on interactive icons and controls.

Use semantic HTML (header, nav, main, section, aside, footer) and ensure keyboard navigation works across components.

Ensure images include descriptive alt attributes; decorative SVGs should have aria-hidden="true".

Offer a prefers-color-scheme fallback if a dark/light mode is implemented in the future.

11. Project Structure & Deliverables

Maintain the previously proposed file structure (src/components, src/pages, src/styles, etc.).

Split UI into reusable components: Header, Footer, Hero, RateCards, ServiceCards, Modules, FAQAccordion, CountryList, MapSection, ChatDrawer, etc.

Include README.md describing how to extend the design system (changing CSS variables for color or spacing).

Document how to disable GSAP animations when exporting HTML to Webflow.
